Singleton - применяется в случаях, когда в программе заведомо используется
	только один объект какого-то класса, для удобства получения ссылок на
	этот объект из любого участка кода.
Abstract Factory - применяется в случае необходимости создавать много схожих
	объектов для обеспечения независимости кода от класса создаваемых объектов,
	позволяет использовать множество конкретных фабрик без привязки к их реализации. 
Builder - используется для конструирования объектов, содержащих большое количество данных.
	Обеспечивает возможность избирательного заполнения полей, убирает необходимость в
	большом количестве перегруженных конструкторов.
Strategy - используется для облегчения последующего расширения программы, убирает зависимости
	одних участков кода от других, исполняемого кода от конкретных реализаций, улучшает
	модульность программы.
